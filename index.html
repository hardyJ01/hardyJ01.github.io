Interdisciplinary Projects and Research

Bitcoin Price Analysis

Project Overview and Objectives

Built ML models to predict Bitcoin price trends using time-series analysis. Applied data preprocessing, visualization, and regression models.

Achieved 85% prediction accuracy in forecasting Bitcoin prices.

Methodologies Employed

Utilized Time-Series analysis to model temporal dependencies and regression models (like Linear Regression, Random Forest) to incorporate external market indicators. This integrated computer science (ML) with financial analysis.

Outcomes and Innovations

The hybrid model demonstrated that while time-series data is crucial, incorporating market sentiment (even if simplified) can improve predictive accuracy. The project resulted in a functional model achieving 85% accuracy on the test set.

Visuals or Prototypes

Generated a series of Matplotlib and Seaborn plots, including price trend decompositions, actual vs. predicted price overlays, and a residual error analysis to validate the model's performance. The final report was delivered as a Jupyter Notebook.

Critical Analysis of Challenges

The primary challenge was the high volatility and non-stationary nature of cryptocurrency data. Overcame this by applying differencing and log transformations. Another challenge was avoiding look-ahead bias during feature engineering, which was solved by using rolling windows.

Python Pandas Matplotlib Seaborn Scikit-learn

Rainfall Prediction

Project Overview and Objectives

Developed a classification model to predict rainfall using weather parameters. Handled data preprocessing, feature selection, and applied algorithms like RNNs and Random Forest.

Achieved high prediction accuracy.

Methodologies Employed

Employed a classification-based approach. The project involved significant data preprocessing and cleaning of meteorological data. We compared several models, including Logistic Regression, K-Nearest Neighbors, and Random Forest, using Scikit-learn.

Outcomes and Innovations

The Random Forest classifier yielded the highest accuracy due to its robustness in handling non-linear relationships between weather parameters. The model's feature importance analysis revealed that 'humidity' and 'wind gust speed' were the most significant predictors.

Visuals or Prototypes

Developed confusion matrices and ROC curves for each model to visually compare their performance. A pair plot was used during exploratory data analysis (EDA) to identify correlations between features.

Critical Analysis of Challenges

Handling missing data points from weather stations was a major hurdle. We addressed this by using K-Nearest Neighbors (KNN) imputation, which proved more effective than simple mean/median imputation for this dataset.

Python Scikit-learn Pandas Matplotlib

# Using a decorator for timing a data-processing function
import time

def timer_decorator(func):
    def wrapper(*args, **kwargs):
        start_time = time.time()
        result = func(*args, **kwargs)
        end_time = time.time()
        print(f"{func.__name__} ran in: {end_time - start_time:.4f}s")
        return result
    return wrapper

@timer_decorator
def process_large_dataframe(df):
    # Complex processing

Global Awareness & Ethical Considerations

In my work, particularly in machine learning, I am increasingly aware of the ethical dimensions of technology. The "move fast and break things" mantra is insufficient when building systems that can impact real people.

Case Study - Predictive Models: In the Rainfall Prediction project, we discussed the ethical implications. A false negative (predicting no rain when a flash flood occurs) could have severe consequences for a farming community. This global issue of climate change and its impact on agriculture was a motivator.

Analysis of Implications: We realized that "accuracy" is not a single metric. We had to decide whether to optimize for 'Recall' (minimizing false negatives) or 'Precision' (minimizing false positives). We chose to optimize for Recall, accepting a few more false alarms as a safer alternative for a high-impact scenario.

Personal Reflection: This taught me that a data scientist's responsibility goes beyond just optimizing a score; it includes understanding the real-world impact, potential biases in the data (e.g., if data is only from wealthy regions), and the potential harm of our models. It's a commitment to building systems that are not only smart but also fair and responsible.
